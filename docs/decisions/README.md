# Architecture Decision Records

> One file per decision. Format: `NNN-short-title.md`
> Status: Proposed | Accepted | Deprecated | Superseded

## Index

| ID | Title | Status | Date |
|----|-------|--------|------|
| — | *(no decisions recorded yet)* | — | — |

## Template

Create `docs/decisions/NNN-title.md`:

```markdown
# NNN — Decision Title

**Status:** Accepted
**Date:** YYYY-MM-DD
**Deciders:** [names or roles]

## Context
What problem were we solving? What constraints existed?

## Decision
What did we decide to do?

## Consequences
What are the trade-offs? What becomes easier/harder?
```

## When to Write an ADR

- Introducing a new architectural pattern
- Choosing between significant technical approaches
- Deprecating or replacing a system component
- Any decision that future agents/developers would otherwise question
